# ТЗ: офлайн-приложение «Где лежит»

## 1. Назначение

Android-приложение для быстрой голосовой фиксации местоположения вещей и последующего поиска. Один пользователь, одно устройство, без сети.

**Пользователь:** взрослый человек, не технарь, пользуется телефоном уверенно, но неохотно осваивает новое. Инструмент должен работать в момент, когда руки заняты вещью.
приложение планирую как просто apk(android), не для google play

## 2. Ключевые ограничения

- Всё выполняется **on-device**. Приложение не запрашивает разрешение INTERNET вообще.
- Путь «нажал → сказал → готово» — не более **5 секунд** и **одного касания** экрана.
- Русский язык, крупный шрифт, высокий контраст.

## 3. Функциональные требования

**F1. Запись**
- Главный экран: одна большая кнопка «Сказать».
- Удержание/нажатие → запись → распознавание через Vosk (`vosk-model-small-ru`, в assets).
- Результат парсится на две части: **предмет** и **место**. Разделитель — предлоги («в», «на», «под», «за»). Первая часть до предлога — предмет, остальное — место.
- Экран подтверждения: два редактируемых поля («ПТС» / «верхний ящик комода в спальне»), кнопки «Сохранить» и «Переписать».
- При неудачном распознавании — поля пустые, ручной ввод доступен всегда.

**F2. Поиск**
- Вторая кнопка «Найти» → голосовой запрос («где документы на машину»).
- Матчинг: нормализация → стеммер → расстояние Левенштейна → словарь синонимов.
- Результат: карточка с предметом, местом и датой записи. Крупно.
- Если совпадений несколько — список по убыванию релевантности, максимум 5.
- Если ноль — показать полный список записей с фильтром.

**F3. Список**
- Все записи, сортировка по дате (новые сверху).
- Свайп влево — удалить. Тап — редактировать.
- Повторная запись того же предмета **перезаписывает** место, старое значение уходит в историю предмета.

**F4. Защита**
- Вход по биометрии/PIN, включается опционально в настройках.
- База — SQLite, зашифрованная (SQLCipher). Ключ в Android Keystore.

## 4. Модель данных

```
Item
  id          INTEGER PK
  name        TEXT     -- "ПТС"
  name_norm   TEXT     -- нормализованная форма для поиска
  location    TEXT     -- "верхний ящик комода в спальне"
  created_at  INTEGER
  updated_at  INTEGER

Synonym
  id          INTEGER PK
  alias_norm  TEXT     -- "документы на машину"
  item_id     INTEGER FK
```

## 5. Нефункциональные требования

| Параметр | Значение |
|---|---|
| Минимальная версия | Android 8.0 (API 26) |
| Размер APK | < 80 МБ (модель Vosk ~45 МБ) |
| Холодный старт | < 1.5 с |
| Распознавание фразы | < 2 с после окончания речи |
| Работа без сети | 100% функционала |
| Разрешения | RECORD_AUDIO, USE_BIOMETRIC. Всё. |

## 6. Стек

Kotlin, Jetpack Compose, Room + SQLCipher, Vosk Android. Сборка — Gradle, подпись собственным ключом. Дистрибуция — APK из GitHub Releases, обновления через Obtainium.

## 7. Что НЕ входит в MVP

Фотографии, категории, комнаты, напоминания, синхронизация, облако, LLM, эмбеддинги, шеринг с семьёй, экспорт. Всё это — только после подтверждения, что базовый сценарий прижился.

## 8. Критерий успеха MVP

Отец за месяц самостоятельно, без напоминаний, сделал **≥ 10 записей** и **≥ 3 успешных поиска**. Если да — продукт живой, можно думать дальше. Если нет — гипотеза не подтвердилась, и никакие фичи из раздела 7 её не спасут.
